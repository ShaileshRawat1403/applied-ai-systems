# 0.4 Learning Log Guide

A key component of this course is tracking your growth as an applied AI practitioner. We do this using the **Learning Log**.

---

## 💾 Setting Up Your Learning Log

1. Navigate to the `templates/` directory in this repository and locate **[learning-log.md](../../templates/learning-log.md)**.
2. Copy this file into your local project workspace (we suggest creating a dedicated `my-workspace/` directory or placing it directly in your workspace root).
3. Open the file and fill out your **Learner Profile** metadata.

---

## 📊 Self-Assessment: Baseline Skill Map

Before starting Module 1, complete the **Baseline Skill Map** table in your log. Be honest with yourself!
* If you have only prompted ChatGPT or Claude but have never written a JSON validation schema, rate your *Prompt Specification Design* as a **1** or **2**.
* If you have read articles about RAG but have never designed a chunking size or metadata schema, rate your *Knowledge Engineering & RAG* as a **1**.
* This baseline helps you measure concrete skill progression. You will re-assess at the mid-point (Module 5) and graduation (Module 9).

---

## 📓 Logging Guidelines

To maintain an effective learning log:
* **Record Time Spent:** Note the duration of each lab. This is useful for planning future pilot deployments.
* **Document Blockers:** Did an API call fail? Did a prompt schema return malformed JSON? Write down how you fixed it. These notes become your personal troubleshooting playbook.
* **Complete Module Reflections:** Each module concludes with three reflection questions. Answering these helps synthesize concepts before moving to the next layer of the model.
