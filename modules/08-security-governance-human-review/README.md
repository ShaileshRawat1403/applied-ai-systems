# Module 8: Security, Governance, and Human Review

## 🎯 Purpose
Design defensive guardrails to protect against prompt injection, secure credential usage, build human approval gates, and document a corporate risk register.

## 📦 Expected Output
- **[Risk Register](templates/risk-register.md)**
- **[Human Review Checklist](templates/human-review-checklist.md)**
- **[Governance Pack](templates/governance-pack.md)**

## 📚 Lessons Outline
1. Security vectors: prompt injection attacks, system leaking, and data exposure.
2. Building secure boundaries: parsing schema inputs and blocking commands.
3. Human-in-the-loop: designing checklists and Slack-integrated approval gates.
4. Compliance & Governance: creating audit trails and system kill switches.
5. Lab: Writing a risk register and security policy for corporate launch.
