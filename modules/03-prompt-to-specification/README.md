# Module 3: Prompt to Specification

## 🎯 Purpose
Transition learners from informal "vibe-based" prompt tweaking to engineering stable, structured, and version-controlled prompt specifications with strict input/output contracts.

## 📦 Expected Output
- **[Prompt Specification Template](templates/prompt-specification.md)**
- **[Output Quality Rubric](templates/output-quality-rubric.md)**

## 📚 Lessons Outline
1. Why prompt engineering is not system design.
2. Formulating Prompt Specifications: inputs, objectives, and JSON schemas.
3. Few-shot example modeling and instruction engineering.
4. Designing objective grading rubrics for model output review.
5. Lab: Drafting your first Prompt Spec and grading rubric.
