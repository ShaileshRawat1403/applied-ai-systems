# Module 7: Evaluation, Debugging, and Observability

## 🎯 Purpose
Move beyond passive output checks. Build programmatic evaluation sets, trace node-by-node execution logs, debug prompt errors, and monitor token consumption costs.

## 📦 Expected Output
- **[Evaluation Harness Template](templates/evaluation-harness-template.md)**
- **[Trace Log Template](templates/trace-log-template.md)**

## 📚 Lessons Outline
1. Why testing AI is different from traditional software: non-determinism.
2. Creating golden evaluation datasets: input scenarios vs. expected facts.
3. System Tracing: capturing prompt-to-response node flow, tokens, and latency.
4. Core debugging techniques: isolating prompt instruction defects.
5. Lab: Formulating an evaluation set and testing execution traces.
