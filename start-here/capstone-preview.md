# Capstone Preview: AI Workflow Pilot Pack

The ultimate goal of this course is to build a comprehensive **AI Workflow Pilot Pack** centered around your selected business use case. This page outlines what this package contains and what is expected of your final submission.

---

## 📂 Pilot Pack Sections

Your final pilot package compiles the templates you complete during the course modules:

1. **Section 1: Opportunity Brief (Module 2)**: The business case, target user boundaries, and success KPIs.
2. **Section 2: Prompt Specification (Module 3)**: Versioned prompt instructions, reasoning limits, and JSON schemas.
3. **Section 3: Output Quality Rubric (Module 3)**: Structural and semantic quality benchmarks.
4. **Section 4: Retrieval Design (Module 4)**: Ingestion chunk sizes, metadata filters, and citation grounding rules.
5. **Section 5: Tool Permission Map (Module 5)**: Allowed function inputs, API schemas, and read/write perimeter blocks.
6. **Section 6: Agent Workflow Canvas (Module 6)**: State variable tracking, multi-agent loop routing, and stop rules.
7. **Section 7: Memory & Context Policy (Module 6)**: Session history buffer sizes and PII anonymization regular expressions.
8. **Section 8: Evaluation Harness (Module 7)**: A programmatic test suite of 20+ edge cases and inputs.
9. **Section 9: Governance Pack (Module 8)**: Corporate risk registers, Slack approval gates, and emergency kill switches.
10. **Section 10: ROI & Demo Script (Module 9)**: Cost-per-run analysis, hours saved estimates, and a 5-minute presentation script.

---

## 📐 The Capstone Quality Bar

A high-quality Capstone Project is:
* **Specific**: The system solves one concrete, narrow task (e.g. drafting changelogs) rather than trying to solve a general department's operations.
* **Inspectable**: Every layer is explicitly documented so that developers can trace where outputs are generated and where gates are run.
* **Safe**: The Control Layer has strict boundaries preventing unauthorized writes and prompt injection attacks.
* **Value-Oriented**: The ROI analysis is grounded in real-world resource times and API token costs.
