# How to Use This Course

This course is designed as a **documentation-first workbook** and learning reference. It is structured to help you *build* rather than just *read*.

---

## 📐 Reusable Lesson Architecture

Every lesson in this book follows the same 10-step template:

1. **Why This Matters**: The real-world business problem and motivation.
2. **Core Idea**: Simple, clear explanation of the core technical concept.
3. **System View**: How the concept maps to the five-layer systems model.
4. **Example**: A realistic case study showing the concept in action.
5. **Practical Walkthrough**: Step-by-step guidance on how to implement it.
6. **Common Failure Modes**: What typically goes wrong and how to fix it.
7. **Lab**: An exercise to test your understanding.
8. **Output**: The specific file or template you need to update.
9. **Review Checklist**: Acceptance criteria to grade your work.
10. **Reflection**: A prompt to help you synthesize what you learned.

---

## 🛠️ Navigating Labs & Templates

1. **The Templates Directory (`templates/`)**: Contains the 20 mandatory course templates in Markdown format. As you progress, you will copy these templates into your personal workspace and fill them out.
2. **The Case Studies Directory (`case-studies/`)**: Contains complete, pre-vetted reference implementations (such as the *AI Change Communication Workflow*) to show you what an expert-level submission looks like.
3. **Your Workspace**: We recommend creating a personal directory (e.g. `my-workspace/`) at the root of this repository. Use this directory to store your completed logs, briefs, and capstone documents. Keep your workspace version-controlled under Git.
